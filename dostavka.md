Внешние скидки в InSales
В SaaS платформе для разработки и эффективной эксплуатации Интернет-магазинов InSales сокрыто множество возможностей для успешных продаж в сети Интернет. Многие возможности требуют определенного исследования и разъяснения.


Нельзя не отметить, что встроенная система для скидок уже включена в большинство тарифных планов InSales и покрывает большую часть потребностей магазина. Кроме того, если встроенной системы скидок не хватает, для вас есть некоторое количество приложений в маркетплейсе интеграций, которые также позволяют делать скидочные программы в случаях, если встроенная система не до конца удовлетворяет вашим потребностям.


Однако, иногда может возникнуть необходимость в создании какой-либо нестандартной скидки именно для вашего магазина. В этой публикации мы расскажем, на конкретном примере, о создании внешних скидок для корзины, которые и призваны решать такие нестандартные задачи.


Что же такое внешняя ссылка? Это некий скрипт, написанный на любом языке программирования. Этот скрипт должен быть размещен на каком-либо сервере и доступен по URL для вашего магазина. Он будет получать от InSales данные корзины в формате JSON и должен будет возвращать ответ в том же формате.


Итак начнём. В административной панели вашего Интернет-магазина внешняя скидка добавляется в разделе Скидки. При добавлении в форму помещается ссылка на ваш скрипт и в общем-то все. Скидка начинает работать в корзине и форме оформления заказа (чекауте).

/**



Файл содержит демонстрационный пример скрипта внешней скидки InSales

@package    InSales-Api

@author     Константин Тарасов info@insales-studio.ru

@category   InSales

@copyright  Copyright (c) 2006-2018 InSales Studio. (https://insales-studio.ru)

@license    MIT License



*/


/**



Сообщения об ошибках
*/
define('NO_ITEMS', 'Товары не обнаружены');
define('LESS_ITEMS', 'Скидка невозможна, поскольку товаров в корзине меньше 2');
define(
'NIL_ITEMS',
'Скидка невозможна, поскольку товаров с ненулевой ценой в корзине меньше 2'
);


/**



Cервисный объект для построения отправляемого в магазин JSON со скидкой
*/
class ResponceDiscount {
public $discount;
public $discount_type;
public $title;


/**



Конcтруктор объект ответа со скидкой

@param $discount      - сумма/процент скидки

@param $discount_type - тип скидки MONEY/PERCENT

@param $discount      - название скидки для отображения в корзине/чекауте
*/
public function __construct($discount = 0, $discount_type = 'MONEY', $title = 'Скидка на самый дешевый товар в корзине') {
$this->title = $title;
$this->discount = $discount;
$this->discount_type = $discount_type;
}
}


/**



Cервисный объект для построения отправляемого в магазин JSON с ошибкой
*/
class ResponceError {
public $errors;


/**



Конcтруктор объект ответа с ошибкой

@param $error_msg - сообщение об ошибке/массив строк с ошибками
*/
public function __construct($error_msg = null) {
$this->errors = array();
if($error_msg) $this->addError($error_msg);
}


public function addError($error_msg) {
if(is_array($error_msg)) {
$this->errors = array_merge($this->errors, $error_msg);
} elseif (is_string($error_msg)) {
$this->errors[] = $error_msg;
}
}
}


function LogEntry($msg) {
file_put_contents('count-discount.log',
sprintf("%s\t%s" . PHP_EOL, date('c'),
(is_scalar($msg) ? $msg : print_r($msg, true))
),
FILE_APPEND
);
}


function process_items($items) {
if(!is_array($items))
return new ResponceError(NO_ITEMS);


$items_count = count($items);
if($items_count < 2)
return new ResponceError(LESS_ITEMS);


// создадим массив цен
$prices = array();
foreach ($items as $item) {
if($item->sale_price == 0)
continue;


$prices[] = $item->sale_price;

}


if(count($prices) < 2)
return new ResponceError(NIL_ITEMS);


// сортируем массив цен товаров в корзине
sort($prices, SORT_NUMERIC);


// создадим объект ответа с минимальной ценой из массива
return new ResponceDiscount($prices[0]);
}


// получаем тело запроса и его отправим сразу в лог
LogEntry($HTTP_RAW_POST_DATA);


// декодируем объект, полученный от InSales
$cart = json_decode($HTTP_RAW_POST_DATA);
// проверим наличие строк корзины в полученном объекте
$items = isset($cart->order_lines) ? $cart->order_lines : null;


$response = process_items($items);


unset($cart, $items);


header('Content-Type: application/json', true, 200);
echo json_encode($response);

Что такое «Внешние скидки» в InSales и как это работает
Суть механизма
Внешние скидки — это способ подключить к InSales свою собственную логику расчёта скидок, которую нельзя реализовать встроенными правилами платформы.

По сути это webhook: InSales при изменении корзины отправляет HTTP-запрос (POST) с JSON-данными корзины на ваш URL, а ваш скрипт должен ответить JSON-ом — какую скидку применить.


InSales (корзина клиента)  →  ваш сервер (скрипт)  →  InSales (применяет скидку)
Архитектура реализации
Компоненты
Магазин InSales — инициирует запрос, отображает результат.
Внешний сервер с вашим скриптом (PHP, Python, Node.js — любой) — получает данные корзины, считает скидку, возвращает ответ.
Настройка в админке InSales — раздел Скидки → Внешняя скидка, где указывается URL вашего скрипта.
Формат обмена
Что приходит от InSales (вход):

json

{
  "order_lines": [
    { "sale_price": 37083, "title": "DELL INSPIRON 5720", ... },
    { "sale_price": 26670, "title": "Телевизор BBK", ... },
    { "sale_price": 491,   "title": "Обработка заказа", ... }
  ],
  "client": { ... },
  "delivery": { ... }
}
Что должен вернуть ваш скрипт (выход):

json

{
  "discount": 491,
  "discount_type": "MONEY",        // или "PERCENT"
  "title": "Скидка на самый дешевый товар"
}
Или ошибка (тогда скидка не применяется):

json

{ "errors": ["Товаров в корзине меньше 2"] }
Пример из статьи
Задача: если в корзине ≥ 2 товаров — дать скидку, равную цене самого дешёвого товара (фактически «3-й/2-й товар в подарок»).

Логика скрипта:

Принять JSON от InSales.
Проверить, что товаров минимум 2.
Отсеять нулевые цены.
Найти минимальную sale_price.
Вернуть её как сумму скидки в формате MONEY.
Путь клиента (CJM) — что видит покупатель
Представим Ивана, который зашёл в магазин электроники.

Шаг 1. Просмотр каталога
Иван выбирает товары. Никакой скидки пока нет — она работает только в корзине.

Шаг 2. Добавление первого товара (ноутбук 37 083 ₽)
Иван открывает корзину.

InSales отправляет POST на скрипт.
Скрипт видит: товар всего 1 → возвращает ошибку «Скидка невозможна, товаров меньше 2».
В корзине скидка не отображается. Иван видит обычный итог.
Шаг 3. Добавление второго товара (телевизор 26 670 ₽)
Иван возвращается в корзину или просто добавляет товар.

InSales снова отправляет JSON со всеми позициями.
Скрипт: товаров ≥ 2, минимальная цена = 26 670 ₽.
Возвращает { "discount": 26670, "discount_type": "MONEY", "title": "Скидка на самый дешевый товар в корзине" }.
В корзине Иван видит:

Ноутбук .......... 37 083 ₽
Телевизор ........ 26 670 ₽
Скидка на самый дешевый товар в корзине: −26 670 ₽
Итого: 37 083 ₽
Шаг 4. Изменение состава
Иван добавляет «Обработку заказа» (491 ₽).

Запрос уходит повторно.
Минимальная ненулевая цена теперь 491 ₽.
Скидка пересчитывается на лету: −491 ₽.
Шаг 5. Оформление заказа (checkout)
Скидка остаётся видимой и на странице оформления. Скрипт может вызываться ещё раз — например, после ввода адреса доставки (потому что в JSON есть данные о доставке и клиенте, и логика может это учитывать).

Шаг 6. Оплата
Клиент платит уже сниженную сумму. Скидка отражается в заказе и в админке как отдельная строка.

Когда это нужно бизнесу
Внешние скидки имеет смысл использовать, когда:

«N-й товар в подарок / самый дешёвый бесплатно»;
персональные скидки из внешней CRM/ERP/лояльности;
скидки по сложным условиям (категория + регион + история покупок);
скидки на основе остатков, времени суток, курса валют;
интеграция с программой лояльности у партнёра.
Что нужно для запуска (чек-лист разработчика)
Сервер с HTTPS и публично доступным URL.
Скрипт-приёмник, который:
читает php://input (или аналог);
парсит JSON;
применяет бизнес-логику;
отдаёт корректный JSON с заголовком Content-Type: application/json.
Логирование входящих запросов — критично для отладки (как в примере с discount.log).
Подключение URL в админке InSales: Скидки → Добавить внешнюю скидку.
Тестирование: положить разные комбинации товаров в корзину и проверить и сумму, и отображаемое название скидки.
Защита: желательно проверять, что запрос приходит именно от InSales (по IP, секретному токену в URL и т.п.), и обрабатывать ошибки/таймауты — скрипт должен отвечать быстро, иначе корзина будет тормозить.
Коротко
Внешняя скидка = ваш микросервис, который в реальном времени решает за InSales, сколько скинуть клиенту. Для клиента это выглядит как обычная строка «Скидка: −XXX ₽» в корзине, которая появляется/пересчитывается автоматически по мере изменения состава покупки. Технически — это простой обмен JSON-ами по HTTP.

Внешние скидки нужны, если магазину не хватает стандартного функционала inSales, и нужно рассчитывать скидки по своему алгоритму.


Для работы внешних скидок потребуется написать и разместить на стороннем сервере скрипт, который будет получать данные о заказе с платформы в формате JSON, обрабатывать их и возвращать ответ в том же формате. То есть со страницы оформления заказа уходят данные о товарах в корзине, вы где-то на стороне их принимаете, считаете, каким должен быть размер скидки, возвращаете это значение, и система его применяет к заказу.

Короткий ответ
Да, такое сделать можно, и «Внешние скидки» — это как раз подходящий инструмент. Это не промокоды (промокодов как отдельной сущности в InSales нет — там «скидки по правилам» и «внешние скидки»). Внешние скидки работают как ваш собственный микросервис, который решает «давать скидку или нет» по любой вашей логике.

Разберу применительно к вашей задаче с игрой и бесплатной доставкой.

Ваш сценарий
«Человек выиграл в игре бесплатную доставку → при следующем заказе она должна автоматически применяться».

Это типовой кейс для внешних скидок. Логика:

У вас есть своя БД (на вашем сервере), где хранится: user_id / email / телефон → у него есть выигранный приз "free_shipping".
Когда клиент оформляет заказ, InSales дёргает ваш скрипт и передаёт JSON корзины (там есть email/телефон клиента + стоимость доставки).
Скрипт ищет клиента в вашей БД:
Есть приз? → возвращает скидку, равную стоимости доставки → доставка фактически бесплатна.
Нет приза? → возвращает errors или скидку 0 → ничего не происходит.
После применения отмечаете приз как «использованный» в своей БД, чтобы он не применился повторно.
Путь клиента
Этап 1. Игра на сайте
Клиент играет, выигрывает «бесплатную доставку».
Вы сохраняете факт выигрыша у себя в БД, привязывая к идентификатору клиента (email, телефон или ID в InSales).
Желательно: сразу показываете ему сообщение «Бесплатная доставка применится автоматически при следующем заказе» — без всяких кодов.
Этап 2. Следующий визит и оформление заказа
Клиент кладёт товары в корзину, идёт в чекаут.
InSales автоматически отправляет POST на ваш скрипт со всеми данными корзины и клиента.
Ваш скрипт:
Берёт email/телефон из JSON.
Проверяет в своей БД: «есть ли у него неиспользованный приз free_shipping?»
Берёт стоимость доставки из JSON.
Возвращает { "discount": <стоимость_доставки>, "discount_type": "MONEY", "title": "Бесплатная доставка за победу в игре" }.
Клиент видит в корзине строку «Бесплатная доставка за победу в игре: −350 ₽».
Этап 3. После оплаты
Через webhook InSales «заказ создан/оплачен» (или через периодический опрос API) вы получаете подтверждение.
Помечаете приз в своей БД как использованный.
При следующих заказах скрипт уже не вернёт скидку.
Важные нюансы именно для бесплатной доставки
1. Внешняя скидка применяется к сумме заказа, а не к доставке напрямую
Это ключевой момент. InSales вернёт скидку — но это будет скидка на итог. Поэтому:

Вы берёте из JSON значение delivery_price (или аналог).
Возвращаете discount = delivery_price в формате MONEY.
Визуально для клиента итог уменьшится ровно на стоимость доставки → эффект «бесплатной доставки».
Если хочется, чтобы строка «Доставка» стала именно 0 ₽, а не «итог минус доставка» — это сложнее и встроенными скидками так не сделать. Обычно ограничиваются вариантом выше (это нормальная практика).

2. Как идентифицировать клиента
В JSON-е заказа есть client.email, client.phone, иногда client.id. На этапе игры вам нужно обязательно знать email или телефон игрока (например, форма перед игрой, или игра доступна только авторизованным).

3. Защита от злоупотреблений
Привязка к email — клиент может создать новый email. Если важно — привязывайте к телефону + подтверждение через SMS, либо к авторизованному аккаунту InSales.
В БД всегда храните статус used: true/false + дату использования.
Что вам понадобится (стек)
Сервер с публичным HTTPS-адресом (VPS, хостинг, Vercel, Railway — что угодно).
Скрипт на любом языке (PHP/Node.js/Python) — принимает POST, читает JSON, идёт в БД, возвращает JSON.
База данных (даже SQLite/JSON-файл на старте сойдёт): таблица prizes(user_email, prize_type, used, won_at, used_at).
Игра на сайте, которая после выигрыша делает запрос к вашему серверу и записывает приз в БД.
InSales API — пригодится, если захотите:
Получать webhook о создании/оплате заказа (чтобы помечать приз использованным).
Подтягивать карточку клиента.
Документация: https://api.insales.ru/
Минимальная схема (псевдокод)
php

// 1. Принимаем запрос от InSales
$cart = json_decode(file_get_contents("php://input"));
$email = $cart->client->email ?? null;
$delivery_price = $cart->delivery_variant->delivery_price ?? 0;

// 2. Проверяем приз в своей БД
$prize = db_find_unused_prize($email, 'free_shipping');

// 3. Отвечаем
header('Content-Type: application/json');
if ($prize && $delivery_price > 0) {
    echo json_encode([
        "discount" => $delivery_price,
        "discount_type" => "MONEY",
        "title" => "Бесплатная доставка за победу в игре 🎮"
    ]);
} else {
    echo json_encode([
        "errors" => [] // тихо ничего не делаем
    ]);
}
Затем отдельным механизмом (webhook от InSales на событие «заказ оплачен») вы вызываете db_mark_prize_used($email).

Ответы на ваши вопросы по пунктам
«здесь речь про скидки, а не про промокоды, верно?»

Да. В InSales нет классических промокодов. Есть:

Встроенные скидки по правилам (категория, сумма, кол-во).

Промокод по сути можно эмулировать через дополнительное поле в чекауте + внешнюю скидку (клиент вводит код → ваш скрипт проверяет его в своей БД). Но вам это не нужно — у вас механика без кода, автоматическая по клиенту.

Можно ли через внешние скидки сделать промокод на бесплатную доставку?
Короткий ответ
Технически — да, можно, но это будет не «настоящий» промокод InSales, а его имитация через механизм внешних скидок + дополнительное поле в чекауте. Работать для клиента будет почти так же.

Разделим понятия
В InSales есть два разных механизма:

1. Встроенные промокоды (раздел «Скидки → Промокоды»)
Создаются в админке.
Клиент вводит код в специальное поле «Промокод» в чекауте.
Дают скидку % или фиксированной суммой на заказ.
Проблема для вас: скидку именно на доставку встроенные промокоды обычно сделать не позволяют — они работают с суммой товаров/заказа. Нужно проверять по вашему тарифу.
2. Внешние скидки (то, что вы изучаете)
Ваш скрипт решает, давать ли скидку.
Поле для ввода кода — это дополнительное поле заказа, которое вы сами создаёте в настройках чекаута.
Можно вернуть любую сумму скидки — в том числе равную стоимости доставки.
Как сделать «промокод на бесплатную доставку» через внешнюю скидку
Схема
В админке: Настройки → Оформление заказа → Дополнительные поля → создаёте поле «Промокод» (текстовое).
В админке: Скидки → Внешние скидки → добавляете URL вашего скрипта и ставите галочку «Зависит от дополнительных полей» (чтобы при вводе кода скрипт пересчитывался).
В шаблоне корзины/чекаута добавляете input для этого поля (как в документации с order[fields_values_attributes]).
У себя на сервере держите БД промокодов: code → тип (free_shipping) → used (true/false) → привязка к клиенту (необязательно).
Скрипт:
Получает JSON, достаёт значение введённого кода и стоимость доставки.
Проверяет код в БД.
Если валиден → возвращает discount = delivery_price, discount_type = "MONEY", title = "Промокод: бесплатная доставка".
Если нет → возвращает errors: ["Промокод не найден или уже использован"].
После оплаты помечаете код использованным (через webhook InSales).
Для клиента это выглядит как обычный промокод
Вводит код → видит «Промокод применён, доставка бесплатно» → итог уменьшается на сумму доставки.
Важный нюанс именно для вашей механики
Вы изначально описывали схему без кода — приз привязан к клиенту автоматически. Это удобнее и безопаснее:

Клиенту не нужно ничего вводить и запоминать.
Код нельзя передать другу/выложить в интернет.
Меньше ошибок и «куда вводить?».
Но если хочется именно «промокод» (например, для маркетинга, чтобы человек чувствовал, что получил «купон»), — делайте через внешнюю скидку + поле, как описано выше. Можно даже совместить: скрипт сначала проверяет привязку к email, и если ничего нет — смотрит введённый код.

Сравнение вариантов
Встроенный промокод	Внешняя скидка с полем «код»	Внешняя скидка по email
Скидка именно на доставку	Сложно/нельзя	✅ Да	✅ Да
Клиент вводит код	✅	✅	❌ (автоматом)
Своя логика (одноразовость, привязка)	Ограниченно	✅ Полностью	✅ Полностью
Нужен свой сервер	❌	✅	✅
Защита от перепродажи кода	Слабая	Средняя	✅ Высокая
Итог
«Настоящие» промокоды InSales для бесплатной доставки — скорее нет (ограниченно).
Через внешние скидки — да, легко, и для клиента это будет неотличимо от обычного промокода.
Для вашей задачи (приз за игру) оптимальнее автоматическая привязка к email/телефону, без ввода кода вообще. Но если хотите «эффект купона» — делайте через внешнюю скидку + дополнительное поле.
